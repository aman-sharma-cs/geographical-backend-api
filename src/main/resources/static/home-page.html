Geographical Data Management System

Simple backend practice project using Spring Boot APIs

About This Project

This project is created to practice backend development concepts like REST APIs, DTOs, services, repositories, and database operations.

The system stores geographical data such as latitude, longitude, description, and remarks for different users.

Available APIs

  • Add Geographical Data
  • Get Geographical Data By User (UUID)
  • Update Existing Geographical Data
  • Delete Geographical Data
  • View All Geographical Data

These APIs are tested using Swagger UI and Postman.

Learning Goals

  • Understanding controller-service-repository flow
  • Working with request and response DTOs
  • Handling HTTP status codes properly
  • Using Swagger for API testing

Contact / Future Improvements

This backend practice project can be extended with authentication, role-based authorization, security configuration, and a full frontend integration.
